# Coinbase Onramp — Guest Checkout Integration

Bloon uses **Coinbase Onramp with Guest Checkout** to let users fund wallets with USDC on Base — no Coinbase account required, no KYC managed by Bloon, and **zero fees on USDC/Base purchases** (with approved access).

---

## Why Coinbase Onramp

| Criteria | Coinbase Onramp | MoonPay | Transak | Ramp Network |
|----------|----------------|---------|---------|--------------|
| USDC on Base fees | **0%** (apply for access) | 1-4.5% | ~1% + spread | ~1-3% |
| Guest Checkout (no account) | US only | No | No | No |
| Apple Pay | Yes | Yes | No | Yes |
| KYC handled by provider | Yes | Yes | Yes | Yes |
| Free to integrate | Yes | Yes | Yes | Yes |
| Embeddable widget/SDK | `<FundCard />`, URL, API | Widget | Widget | Widget |
| Sandbox/testing | Yes | Yes | Yes | Yes |

**Key advantage:** 0% fees for USDC on Base. Requires approval at [coinbase.com/developer-platform/developer-interest](https://www.coinbase.com/developer-platform/developer-interest). Standard fees (without 0% program) are 3-5%.

## Guest Checkout

For users **not logged into a Coinbase account**. Automatically directed to this flow.

### Eligibility & Limits

- **Who:** Any US resident with a debit card
- **Payment:** Debit card, Apple Pay (debit card in Apple Pay wallet)
- **Weekly limit:** $500/week (may be higher for partner apps)
- **Minimum:** $5
- **Currency:** USD only
- **No Coinbase account required**

### What Coinbase Handles

- KYC/identity verification (lightweight, via payment method)
- Fraud detection and chargeback claims
- Payment processing (card, Apple Pay)
- Regulatory compliance
- Customer support for payment issues

### User Flow

```
User opens Bloon funding page (/fund/:token)
  -> Sees Coinbase Onramp widget (embedded)
  -> Not logged into Coinbase -> enters Guest Checkout automatically
  -> Enters debit card OR taps Apple Pay
  -> Enters phone number (US) for verification
  -> Confirms purchase amount
  -> USDC sent directly to Bloon wallet address on Base
  -> Bloon detects deposit on-chain -> wallet funded
```

With Apple Pay, the entire flow takes **seconds**.

## Integration Options

### Option 1: One-Click-Buy URL (Recommended for Bloon)

Generate a pre-filled URL server-side. If user has no Coinbase session, lands in Guest Checkout automatically. Works with any frontend (no React dependency).

**How it works:**

1. Get a CDP API key from [portal.cdp.coinbase.com](https://portal.cdp.coinbase.com)

2. Generate a session token (server-side):

```bash
curl -X POST 'https://api.developer.coinbase.com/onramp/v1/token' \
  -H 'Authorization: Bearer <CDP_JWT>' \
  -H 'Content-Type: application/json' \
  -d '{
    "addresses": [{
      "address": "0x<BLOON_WALLET_ADDRESS>",
      "blockchains": ["base"]
    }],
    "assets": ["USDC"]
  }'
```

3. Construct the One-Click-Buy URL:

```
https://pay.coinbase.com/buy/select-asset
  ?sessionToken=<token>
  &defaultAsset=USDC
  &fiatCurrency=USD
  &presetFiatAmount=50
  &defaultPaymentMethod=CARD
```

4. Redirect user or open in iframe/popup on the funding page.

**URL Parameters:**

| Parameter | Required | Description |
|-----------|----------|-------------|
| `sessionToken` | Yes | Generated by Session Token API |
| `defaultAsset` | Yes | Set to `USDC` |
| `presetFiatAmount` | Yes* | Amount in USD (e.g., `50`) |
| `fiatCurrency` | Yes* | Set to `USD` |
| `defaultPaymentMethod` | No | `CARD` to default to debit card |

*Either `presetFiatAmount` + `fiatCurrency` or `presetCryptoAmount` is required.

**Important:** Session tokens are **single-use** and expire quickly. Generate a fresh one per funding session.

### Option 2: `<FundCard />` React Component

Pre-built React component from Coinbase OnchainKit. Handles entire on-ramp UI.

```bash
npm install @coinbase/onchainkit
```

```tsx
import { FundCard } from '@coinbase/onchainkit/fund';
<FundCard />
```

Requires CDP Project ID + OnchainKit API Key. Only useful if funding page becomes a React app (not current Bloon architecture).

### Option 3: Create Onramp Session API

Single API call returns both a quote and a ready-to-use URL:

```bash
curl -X POST 'https://api.developer.coinbase.com/onramp/v2/session' \
  -H 'Authorization: Bearer <CDP_JWT>' \
  -H 'Content-Type: application/json' \
  -d '{
    "destinationAddress": "0x<BLOON_WALLET_ADDRESS>",
    "destinationNetwork": "base",
    "purchaseCurrency": "USDC",
    "paymentAmount": "50.00",
    "paymentCurrency": "USD",
    "paymentMethod": "CARD",
    "country": "US",
    "subdivision": "CA"
  }'
```

Returns `session.onrampUrl` + `quote` (pricing breakdown).

## Deposit Detection

After the user completes the Onramp flow, Bloon detects the deposit:

**Option A: On-chain monitoring (recommended for v1)**
- Watch for USDC transfers to the wallet address via viem + Base RPC
- Works for both Onramp deposits AND direct transfers
- Simplest — Bloon already has this infrastructure (balance polling on funding page)

**Option B: Webhooks (future)**
- Subscribe to Coinbase Onramp webhooks for real-time status updates
- HMAC-signed for verification
- Better UX (instant confirmation vs polling delay)

**Option C: Poll Transaction Status API**
- Use `partnerUserRef` to link transactions to users
- Poll after onramp URL is visited with exponential backoff

## Implementation in Bloon

### What Changes

| Component | Change |
|-----------|--------|
| `packages/api/src/routes/fund.ts` | Add endpoint to generate Coinbase session token |
| Funding page HTML | Embed Onramp iframe/redirect alongside existing QR code |
| `.env` | Add `CDP_API_KEY_NAME`, `CDP_API_KEY_SECRET` |
| `.env.example` | Document new CDP env vars |
| `packages/core/src/config.ts` | Add CDP config accessors |

### What Stays the Same

- Wallet creation (`/api/wallets`)
- Buy/confirm flow
- On-chain USDC balance monitoring
- QR code (still available as fallback)
- Everything in core, wallet, x402, checkout, orchestrator

### New Endpoint

```
GET /fund/:token/onramp-session
  -> Generates Coinbase session token for this wallet's address
  -> Returns { onrampUrl: "https://pay.coinbase.com/buy/..." }
  -> Funding page JS calls this, then opens the URL in iframe or redirect
```

### Security

- Backend generates session tokens (CDP API keys never exposed to client)
- Backend signs JWT with CDP API keys using ES256
- Session tokens are single-use and short-lived
- No user identity data stored by Bloon

## Sandbox Testing

**Widget:** Open Onramp modal -> at "Secured by Coinbase" footer -> click "Secured" 10 times -> Debug Menu -> Actions tab -> toggle "Enable Mocked Buy and Send" -> exit Debug Menu -> mock transactions work.

**API:** Prefix `partnerUserRef` with `sandbox-` (e.g., `sandbox-user-1234`). Sandbox orders succeed without charging any card.

**Apple Pay on localhost:** Append `&useApplePaySandbox=true` to payment link URL.

## Limitations

- **US only** for Guest Checkout (non-US users need a Coinbase account)
- **$500/week limit** per user (may be negotiable for partners)
- **$5 minimum** transaction
- **USD only** for Guest Checkout
- **Debit cards only** (no credit cards)
- **No offramp** via Guest Checkout (cashing out requires full Coinbase account)
- **WebView limitations** — URLs don't work in native WebView/WKWebView (no Passkeys/WebAuthn support). Use Chrome Custom Tabs or SFSafariViewController on mobile.
- **Apple Pay web** requires additional setup: domain registration, verification file, coordination with Coinbase
- **0% USDC fees require approval** — standard fees are 3-5% without it
