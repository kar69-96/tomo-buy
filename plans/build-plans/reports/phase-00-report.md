## Deviations from the plan
- **`RoutingDecision.explain_cant` vs runbook `explain?`** — used `explain_cant` (data-models §6 full
  field list) instead of the runbook's shorthand `explain?`. **Wave-2 inherits `explain_cant`.**
- **`Path` includes `EXPLAIN_CANT` as a member** — per runbook line 78's union, so `RoutingDecision.path`
  is typed `Path` (which contains `EXPLAIN_CANT`), equivalent to §6's `Path | "EXPLAIN_CANT"`.
- **`TomoError` base class** — the runbook requires a base hierarchy; AgentPay's errors are flat. Added
  the base (abstract `code`, `name` via `new.target`, prototype-chain restore) — one intentional
  departure from the ported AgentPay shape.
- **Data types derived via `z.infer`** — schemas are the single source of truth (§7 anti-drift); the
  `types/*.ts` files re-export the inferred types rather than declaring parallel `interface`s.
- **Coverage excludes `src/types/**`** — those are type-only re-export files (zero runtime code).
